# 기본상품명에서 핵심 키워드 추출
상품명에서 핵심 키워드를 추출하는 과정은 검색 최적화의 중요한 부분
상품명을 분석하여 메인 키워드(주제어)와 보조 키워드(설명어)를 자동으로 추출하는 방법이 필요
GPT를 활용하여 사전을 만들기로 함 

초기 작업에서는 모든 상품명에 대해 GPT API를 호출하지 않고도 
효율적으로 메인 키워드를 추출하고 사전을 구축하는 방법이 있으며
이를 통해 API 호출 횟수를 줄이고 비용을 절감 가능

## 세부 단계
1. 대표 상품군 선정

전체 상품명 리스트에서 자주 등장하는 주요 상품군을 대표로 선정
예를 들어, ‘의자’, ‘옷걸이’, ‘스탠드’, ‘텐트’ 등과 같은 상품군이 자주 나타난다면 이를 대표 상품군으로 설정
이런 식으로 최소한의 범위로 전체 상품군을 대표하는 키워드 목록을 만들면, 
모든 상품명을 처리하지 않아도 대다수의 키워드를 사전 구축에 반영할 수 있습니다.


2. 대표 상품군에 대해 GPT API 호출
선택된 대표 상품군의 상품명에 대해서만 GPT API를 호출해 메인 키워드를 추출
이때 API를 통해 추출된 대표 상품군 키워드(예: 의자, 옷걸이)와 
그에 따른 관련 키워드(예: 접이식, 감성, 경량)를 사전에 수집
예를 들어, ‘캠핑용 접이식 의자’와 ‘경량 휴대용 의자’ 같은 여러 상품명을 넣어두면 
GPT가 공통된 핵심 키워드로 의자를 제시해 줄 것임

3. 사전 구축 및 확장

대표 상품군 키워드를 바탕으로 상품명 키워드 사전을 구축
사전에는 주요 상품군 키워드(의자, 옷걸이 등)와 연관된 서브 키워드(접이식, 휴대용, 감성 등)를 추가하여 활용
이후 새로운 상품명이 들어올 때 사전과 매칭하여 키워드를 추출
이렇게 하면 GPT API 호출 없이도 새 상품명을 빠르게 처리할 수 있음


4. 사전 기반 키워드 추출 로직 구축

사전이 구축되면, 사전에서 매칭되는 단어가 있으면 해당 키워드를 메인으로 자동 선택하고, 
매칭되지 않는 경우에만 예외적으로 GPT API를 호출하여 메인 키워드를 추출하는 방식으로 비용을 최적화할 수 있음
예를 들어, 캠핑용 접이식 의자라는 상품명이 들어왔다면 다음과 같이 처리됩니다:
- 사전 매칭: 사전에 등록된 대표 키워드 의자가 상품명에 포함되어 있으므로 바로 의자를 메인 키워드로 추출.
- 연관 서브 키워드 매칭: 추가적으로 접이식, 캠핑용 등의 서브 키워드도 사전에 있으면 이를 보조 키워드로 활용.

5. 최종 목표
GPT 호출 최소화: 사전으로 처리 가능한 상품명은 GPT 호출을 하지 않도록 하여 비용 절감.
대표 상품군 기반의 사전 확장: 사전이 점차 풍부해지면서 모든 상품명을 GPT 없이도 사전으로 처리하도록 하는 것이 최종 목표
