# 📦 최적화 자동화 학습용 데이터 정규화 설계

## 목적
현재 수작업으로 진행 중인 "후속 최적화 작업"의 결과물들을 정형화된 데이터로 누적하여,
향후 **상품명 자동화 2.0**의 학습 데이터셋으로 활용하기 위한 기반을 마련한다.

---

## 📁 데이터 수집 항목 (엑셀 구조 예시)

| 구분 | 항목명 | 설명 |
|------|--------|------|
| A | 기존상품명 | 도매처 제공 상품명 또는 최초 등록 상품명 |
| B | 메인 키워드 | 이 상품을 대표하는 핵심 키워드 (예: 남성정장바지) |
| C | 고정 키워드 | 필수로 들어가야 할 키워드들 (옵션, 특징 등) |
| D | 최적화 상품명 | 내가 수작업으로 만든 최종 상품명 |
| E | 네이버 상위 키워드 | 상위판매자 분석을 통해 추출된 키워드 리스트 |
| F | 연관 검색어 | 연관검색어 API 기반 추출 키워드 |
| G | 검색 태그 | 실제로 등록한 태그 목록 |
| H | 카테고리 트리 | "카테고리1 > 카테고리2 > ..." 형태로 수집 |
| I | 판매 이력 | 리뷰 수 또는 판매 수 등 (선택) |
| J | 전환 키워드 | 실제 유입에 기여했을 것으로 추정되는 키워드 (후보군) |

---

## 🧠 향후 자동화 설계 기반

### 1. 상품명 조합 알고리즘
- `[메인 키워드] + [상위 키워드 2~3개] + [고정 키워드] → 최적화 상품명 추천`

### 2. 태그 추천 모델
- 최적화 상품명 → 상위 CTR/조회수 기반으로 유사 태그 자동 추천

### 3. 카테고리-키워드 매칭 평가
- 카테고리별로 어떤 키워드가 자주 등장했는지 자동 분석 → 매칭률 계산

### 4. 유입 키워드 예측
- 실제 유입이 발생한 키워드 기반으로 머신러닝/룰 기반 예측 모델 구성

---

## ✨ 활용 방향
- 초기에는 엑셀 수기로 데이터 누적
- 50~100개 단위로 분석하여 반복되는 키워드 패턴 수집
- 이후 구조화된 데이터를 기반으로 Python, GPT, 또는 Custom Rule Engine 등으로 자동화

---

## ✅ 요약
> 지금의 수작업은 단순한 정성 작업이 아닌,
> 미래 자동화를 위한 정답 데이터셋 수집 과정이다.

이 설계를 바탕으로 **“팔리는 상품명 조합”**을 학습한 내 자동화 알고리즘을 만들 수 있다.

# 🤖 스마트스토어 최적화 자동화를 위한 종합 설계 전략

## 1. 목적
- 후속 최적화 작업을 기반으로, 향후 **자동화 2.0** 시스템 구축을 위한 전략 정리
- 실제로 팔리는 상품명과 태그 구성 → 반복 가능한 자동화 구조화

---

## 2. 데이터 수집 및 저장 전략

### ✅ 기본 저장 항목 (상품별 1행)

| 항목 | 설명 |
|------|------|
| 기존 상품명 | 도매처 제공 또는 등록 초기 상품명 |
| 메인 키워드 | 상품군을 대표하는 키워드 (예: 양면테이프) |
| 고정 키워드 | 상품 구성요소나 필수 정보 (예: 3세트, 1봉 등) |
| 최적화 상품명 | 수작업으로 구성한 판매 중심 상품명 |
| 검색 태그 | 직접 입력한 태그 (CTR/조회수 기반 구성) |
| 카테고리 트리 | category1~4의 조합 + 전체 문자열 형태도 함께 저장 |
| 혼합형 키워드 | 빈도우위 키워드 기반 조합 참고용 (브랜드 제거 필요) |
| 롱테일 키워드 | 빈도하위 키워드 기반 조합 참고용 (브랜드 제거 필요) |

---

## 3. 브랜드 키워드 관련 주의사항
- 네이버 상위 키워드/연관검색어에는 브랜드명이 포함된 경우가 많음
- 따라서 브랜드 키워드는 상품명 조합에 사용하지 않고, **참고용 또는 필터링 대상으로만 저장**

---

## 4. 자동화 설계를 위한 활용 전략

### ✅ 1) 키워드 조합 기반 상품명 생성
- 최적화 키워드(메인/고정 제외) + 기존 상품명 키워드에서 자주 등장한 키워드 → 빈도 기반 조합
- 브랜드 키워드는 제외하고 의미상 어색하지 않은 단어만 자동 선택

### ✅ 2) 카테고리 기반 키워드 추천
- 각 카테고리별로 등장한 키워드들을 누적 집계 → 키워드 뱅크 형성
- 새로운 상품이 해당 카테고리에 속할 경우 → 자주 쓰인 키워드 추천 가능

### ✅ 3) 태그 추천 자동화
- 검색태그에 쓰인 키워드들을 누적하여 CTR/조회수 기반으로 우선순위 설정
- 상품명과 연계하여 자동 태그 10개 추천 가능

### ✅ 4) 최적화 상품명 아카이빙
- 만들어낸 최적화 상품명 그 자체를 **향후 자동화 추천 후보군**으로 활용
- 의미 분류가 어렵더라도, 실제 팔렸던 표현 자체를 다시 써먹을 수 있음

---

## 5. 자동화 구현 순서 제안

1. 상품별 데이터 정리 (엑셀 기반 수집 지속)
2. 상품명 키워드/태그/카테고리 데이터 누적
3. 카테고리별 자주 쓰인 키워드 집계 → 키워드뱅크 구축
4. 키워드 조합 알고리즘 설계 (초기엔 룰 기반, 이후 GPT 적용 가능)
5. 저장된 최적화 상품명 샘플 기반 → 자동 추천 후보군 생성기 도입
