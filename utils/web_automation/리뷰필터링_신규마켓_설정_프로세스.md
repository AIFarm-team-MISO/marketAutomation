# 스마트스토어 리뷰필터링 신규 마켓 설정 및 실행 프로세스

작성일: 2026-07-07  
프로젝트 경로: `F:\marketAutomation`  
작업 목적: 마켓별 리뷰 엑셀과 비교 대상 상품 파일을 매칭하여 리뷰 보유 상품을 빠르게 분류한다.

---

## 1. 목적

스마트스토어 리뷰 다운로드 엑셀을 기준으로, 비교 대상 상품 파일의 상품번호와 매칭하여 아래 기준으로 상품을 분류한다.

```text
리뷰있음
리뷰없음
리뷰3개이상
리뷰1~2개
```

이 결과는 이후 아래 작업의 기준 데이터로 사용한다.

- 리뷰상품 선별
- 그룹상품 전환 후보 선정
- 단독 리터치 후보 선정
- 판매유지 / 보류 / 삭제검토 판단
- 마켓별 1차 구조전환 작업

---

## 2. 전체 작업 흐름

```text
1. 마켓키 결정
2. 리뷰 원본 폴더 수동 생성
3. 리뷰 엑셀 파일 다운로드
4. 리뷰 엑셀을 마켓 폴더에 넣기
5. 설정파일에서 현재 작업 마켓 선택
6. 비교 대상 상품 파일 준비
7. 비교 대상 상품 파일을 리뷰필터링 폴더에 넣기
8. 프로그램 실행
9. master 파일 생성 확인
10. 결과 엑셀 확인
```

---

## 3. 마켓키 결정

먼저 코드에서 사용할 마켓키를 정한다.

예시:

```text
원스톱리빙 → onestop_living
오늘담음 → today_dameum
```

마켓키는 아래 항목에 함께 사용된다.

- 리뷰 원본 폴더명
- master 파일명
- 설정파일의 마켓키
- 로그 출력
- 결과 파일명 일부

마켓키 추천 규칙:

```text
영문 소문자 사용
띄어쓰기 없음
언더바 사용
마켓마다 고유한 이름 사용
중간에 변경하지 않기
```

## 4. 리뷰 원본 폴더 수동 생성

리뷰 원본 폴더는 코드에서 자동 생성하지 않는다.  
사용자가 직접 만든다.

폴더 위치:

```text
F:\marketAutomation\utils\web_automation\smartstore_review_checker\review_data\{market_name}
```

예시:

```text
F:\marketAutomation\utils\web_automation\smartstore_review_checker\review_data\onestop_living
F:\marketAutomation\utils\web_automation\smartstore_review_checker\review_data\today_dameum
```

주의:

```text
review_data/{market_name} 폴더는 직접 만든다.
review_data/{market_name}/master 폴더는 코드가 자동 생성한다.
```

이렇게 하는 이유:

```text
마켓키 오타로 잘못된 폴더가 자동 생성되는 것을 막기 위해서다.
예: today_dameum을 today_damum으로 잘못 입력했을 때 실수를 바로 확인하기 위함.
```

## 5. 리뷰 엑셀 파일 준비

스마트스토어센터에서 해당 마켓의 리뷰 엑셀을 기간별로 다운로드한다.

예시 파일명:

```text
review_202001-202012.xlsx
review_202101-202112.xlsx
review_202201-202212.xlsx
review_202301-202312.xlsx
review_202401-202412.xlsx
review_202501-202512.xlsx
review_202601-202606.xlsx
```

파일명은 반드시 위 형식일 필요는 없지만, 기간이 보이도록 정리하는 것이 좋다.

## 6. 리뷰 엑셀을 마켓 폴더에 넣기

다운로드한 리뷰 엑셀 파일을 마켓별 리뷰 원본 폴더에 넣는다.

예시:

```text
F:\marketAutomation\utils\web_automation\smartstore_review_checker\review_data\today_dameum\review_202401-202412.xlsx
F:\marketAutomation\utils\web_automation\smartstore_review_checker\review_data\today_dameum\review_202501-202512.xlsx
F:\marketAutomation\utils\web_automation\smartstore_review_checker\review_data\today_dameum\review_202601-202606.xlsx
```

주의:

```text
리뷰 원본 폴더에는 스마트스토어 리뷰 다운로드 원본만 넣는다.
리뷰필터링 결과 파일은 넣지 않는다.
master 파일은 넣지 않는다.
임시 엑셀 파일(~$로 시작하는 파일)은 없어야 한다.
```

## 7. 설정파일에 마켓 정보 등록

설정파일 위치:

```text
F:\marketAutomation\config\web_automation_settings.py
```

설정 예시:

```python
# ============================================================
# 스마트스토어 리뷰 필터링 설정
# ============================================================

SMARTSTORE_REVIEW_ACTIVE_MARKET = "today_dameum"
# SMARTSTORE_REVIEW_ACTIVE_MARKET = "onestop_living"

SMARTSTORE_REVIEW_MARKETS = {
    "onestop_living": {
        "market_name": "onestop_living",
        "market_display_name": "원스톱리빙",
    },
    "today_dameum": {
        "market_name": "today_dameum",
        "market_display_name": "오늘담음",
    },
}


def get_smartstore_review_market_config(market_key=None):
    """
    스마트스토어 리뷰 필터링용 마켓 설정을 반환한다.

    market_key를 따로 넘기지 않으면
    SMARTSTORE_REVIEW_ACTIVE_MARKET 값을 기준으로 현재 작업 마켓을 선택한다.
    """

    if market_key is None:
        market_key = SMARTSTORE_REVIEW_ACTIVE_MARKET

    if market_key not in SMARTSTORE_REVIEW_MARKETS:
        raise ValueError(
            f"등록되지 않은 스마트스토어 리뷰 필터링 마켓입니다: {market_key}"
        )

    return SMARTSTORE_REVIEW_MARKETS[market_key]
```

현재 실행할 마켓은 아래 값으로 선택한다.

```python
SMARTSTORE_REVIEW_ACTIVE_MARKET = "today_dameum"
```

원스톱리빙으로 다시 실행할 경우:

```python
SMARTSTORE_REVIEW_ACTIVE_MARKET = "onestop_living"
```

## 8. 비교 대상 상품 파일 준비

리뷰와 매칭할 비교 대상 상품 파일을 준비한다.

필수 컬럼:

```text
상품번호(스마트스토어)
판매자상품코드
```

있으면 좋은 컬럼:

```text
그룹상품번호
상품명
스마트스토어전용 상품명
채널
```

비교 대상 상품 파일은 스마트스토어 상품조회/수정에서 받은 상품 파일 또는 가공된 상품 리스트 파일을 사용한다.

## 12. 결과 엑셀 시트 구성

결과 파일에는 아래 시트가 생성된다.

```text
00_요약
01_전체비교결과
02_리뷰있음
03_리뷰없음
04_리뷰3개이상
05_리뷰1~2개
```

확인 순서:

```text
1. 00_요약
2. 04_리뷰3개이상
3. 05_리뷰1~2개
4. 02_리뷰있음
5. 03_리뷰없음
```

## 13. 결과 해석 기준

### 리뷰3개이상

우선 검토 대상이다.

```text
리뷰 자산이 어느 정도 있는 상품
그룹화 대표상품 후보
단독 리터치 후보
광고 테스트 후보
```

---

### 리뷰1~2개

보조 검토 대상이다.

```text
리뷰는 적지만 고객 반응이 있었던 상품
최근 판매 여부와 함께 판단
그룹 가능성이 있으면 2순위 후보
```

---

### 리뷰있음

리뷰 수와 관계없이 리뷰가 있는 전체 상품이다.

```text
리뷰상품 전체 확인용
그룹화 가능 여부 추가 판단
단독 유지 여부 판단
```

---

### 리뷰없음

현재 리뷰 데이터상 매칭되지 않은 상품이다.

```text
리뷰 없음
최근 판매 여부, 마진, 그룹 가능성에 따라 판단
무판매·무리뷰·그룹불가 상품은 보류 또는 삭제검토
```

## 14. 오류별 확인 방법

### 오류 1. 리뷰 원본 폴더 없음

예상 메시지:

```text
[리뷰 원본 폴더 없음]
```

확인할 것:

```text
review_data/{market_name} 폴더를 직접 만들었는가
설정파일의 market_name과 폴더명이 정확히 같은가
오타가 없는가
```

---

### 오류 2. 리뷰 원본 파일 없음

예상 메시지:

```text
[리뷰 원본 파일 없음]
```

확인할 것:

```text
마켓 폴더 안에 리뷰 엑셀 파일이 있는가
파일 확장자가 .xlsx 또는 .xls 인가
결과 파일이나 master 파일만 들어 있는 것은 아닌가
```

---

### 오류 3. 비교 대상 상품 폴더 안에 읽을 파일이 없음

예상 메시지:

```text
ValueError: 비교 대상 상품 폴더 안에 읽을 파일이 없습니다.
```

확인할 것:

```text
F:\work\#쇼핑몰\#대량등록\#상품순환 엑셀파일\6.리뷰필터링파일
위 폴더 안에 비교 대상 상품 파일이 있는가

결과 파일만 남아 있지는 않은가
Product_*.csv 또는 .xlsx 파일이 있는가
```

---

### 오류 4. 필수 컬럼 없음

예상 원인:

```text
비교 대상 상품 파일에 필수 컬럼이 없음
```

필수 컬럼:

```text
상품번호(스마트스토어)
판매자상품코드
```

## 15. 신규 마켓 설정 체크리스트

```text
□ 마켓키를 정했다.
□ review_data/{market_name} 폴더를 직접 만들었다.
□ 리뷰 엑셀 파일을 마켓 폴더에 넣었다.
□ 설정파일에 마켓 정보를 추가했다.
□ SMARTSTORE_REVIEW_ACTIVE_MARKET 값을 현재 마켓으로 바꿨다.
□ 비교 대상 상품 파일을 리뷰필터링파일 폴더에 넣었다.
□ 비교 대상 상품 파일에 상품번호(스마트스토어), 판매자상품코드 컬럼이 있다.
□ 실행했다.
□ master 파일이 생성되었다.
□ 결과 엑셀이 생성되었다.
□ 00_요약을 확인했다.
□ 리뷰있음 / 리뷰없음 분류를 확인했다.
□ 리뷰3개이상 / 리뷰1~2개 시트를 확인했다.
```
