# 도매처 품절필터링 실행 프로세스

작성일: 2026-07-07  
프로젝트 경로: `F:\marketAutomation`  
현재 대상 도매처: 3MRO  
작업 목적: 도매처 상품코드를 기준으로 상품 상태를 자동 확인하여 품절, 제품없음, 확인필요 상품을 분류한다.

---

## 1. 목적

도매처 사이트에서 상품코드를 검색하여 현재 상품 상태를 자동 판별한다.

분류 기준:

```text
정상
품절
제품없음
확인필요
확인실패
```

이 결과는 마켓 상품의 품절처리, 판매중지, 삭제검토, 수동확인 작업의 기준으로 사용한다.

---

## 2. 전체 작업 흐름

```text
1. 확인할 도매처 결정
2. 상품코드 준비
3. input_product_codes.txt 파일 열기
4. 상품코드 붙여넣기
5. 입력 파일 저장
6. 품절 확인 프로그램 실행
7. 결과 파일 확인
8. 결과를 마켓 관리파일에 반영
```

---

## 3. 현재 자동화 대상

현재 1차 사용 가능한 대상 도매처:

```text
3MRO
```

현재 3MRO 자동화 가능 작업:

```text
사이트 접속
자동 로그인
상품코드 검색
검색 결과 상품번호 확인
품절 표시 확인
정상 / 품절 / 제품없음 / 확인필요 / 확인실패 분류
결과 텍스트 파일 저장
```

---

## 4. 상품코드 준비

엑셀에서 확인할 상품코드 열을 복사한다.

예시:

```text
20230
20299
20325
20328
20527
```

주의:

```text
판매자상품코드 전체가 아니라 도매처 상품코드만 넣는다.
```

예시:

```text
SP-3MR_20529  → X
20529         → O
```

---

## 5. 입력 파일 위치

상품코드 입력 파일:

```text
F:\marketAutomation\utils\web_automation\input_product_codes.txt
```

이 파일을 열고 기존 내용을 지운 뒤 확인할 상품코드를 붙여넣는다.

예시:

```text
20230
20299
20325
20328
20527
```

---

## 6. 입력 파일 저장 전 확인

저장 전 확인할 것:

```text
상품코드 앞뒤에 공백이 없는가
빈 줄이 너무 많이 들어가지 않았는가
판매자상품코드가 아니라 도매처 상품코드만 들어갔는가
엑셀 헤더명까지 같이 복사하지 않았는가
```

---

## 7. 프로그램 실행

터미널에서 실행한다.

현재 실행 파일이 `web_automation.py` 기준이라면 아래처럼 실행한다.

```powershell
& F:/marketAutomation/myenv/Scripts/python.exe f:/marketAutomation/utils/web_automation/web_automation.py
```

실행 흐름:

```text
1. 3MRO 사이트 접속
2. 로그인 상태 확인
3. 필요 시 자동 로그인
4. input_product_codes.txt 읽기
5. 상품코드별 검색
6. 검색 결과에서 NO.상품번호 확인
7. 품절 표시 확인
8. 정상 / 품절 / 제품없음 / 확인필요 / 확인실패 분류
9. 콘솔에 결과 출력
10. product_status_result.txt 파일 저장
```

## 9. 상품 상태 분류 기준

### 정상

판정 기준:

```text
검색한 상품코드와 정확히 일치하는 NO.상품번호가 있고,
해당 상품 카드 안에 품절 표시가 없는 경우
```

예시:

```text
검색 코드: 20697
검색 결과에 NO.20697 있음
품절 표시 없음
→ 정상
```

상태값:

```text
selling
```

처리:

```text
판매 유지 가능
```

---

### 품절

판정 기준:

```text
검색한 상품코드와 정확히 일치하는 NO.상품번호가 있고,
해당 상품 카드 안에 품절 표시가 있는 경우
```

예시:

```text
검색 코드: 11221
검색 결과에 NO.11221 있음
상품 카드 안에 품절 표시 있음
→ 품절
```

상태값:

```text
sold_out
```

처리:

```text
마켓 상품 품절처리 또는 판매중지 검토
```

---

### 제품없음

판정 기준:

```text
검색 결과 자체가 없고,
NO.상품번호 후보도 없는 경우
```

예시:

```text
검색 코드: 20695
검색 결과 개수 0
NO.상품번호 후보 없음
→ 제품없음
```

상태값:

```text
not_found
```

처리:

```text
도매처 상품 삭제 가능성 높음
마켓 상품 판매중지 또는 삭제검토
```

---

### 확인필요

판정 기준:

```text
검색 결과는 있지만,
검색한 상품코드와 정확히 일치하는 NO.상품번호가 없는 경우
```

예시:

```text
검색 코드: 203
검색 결과에 NO.20325, NO.20328 등은 있음
하지만 정확히 NO.203은 없음
→ 확인필요
```

상태값:

```text
need_check
```

처리:

```text
수동 확인 필요
짧은 코드나 유사 상품이 많이 검색되는 경우 주의
```

---

### 확인실패

판정 기준:

```text
사이트 로딩 오류
로그인 오류
검색 오류
selector 오류
일시적인 Selenium 오류
```

상태값:

```text
failed
```

처리:

```text
다시 실행하거나 수동 확인
```

## 10. 결과 반영 기준

결과를 마켓 관리파일 또는 작업 엑셀에 반영한다.

추천 처리:

```text
정상 → 유지
품절 → 품절처리 / 판매중지
제품없음 → 삭제검토 / 판매중지
확인필요 → 수동확인
확인실패 → 재실행
```

---

## 11. 수동 확인이 필요한 경우

아래 경우는 자동 판정만 믿지 않고 수동 확인한다.

```text
확인필요로 분류된 상품
확인실패로 분류된 상품
상품코드가 너무 짧은 상품
검색 결과가 여러 개 나오는 상품
도매처 상품명과 마켓 상품명이 크게 다른 상품
리뷰나 판매이력이 있는 중요 상품
최근 판매된 상품
고마진 상품
그룹상품 대표상품 후보
```

## 12. 품절필터링 작업 체크리스트

```text
□ 확인할 도매처를 정했다.
□ 도매처 상품코드만 추출했다.
□ input_product_codes.txt 파일을 열었다.
□ 기존 내용을 지웠다.
□ 상품코드를 붙여넣었다.
□ 입력 파일을 저장했다.
□ 품절 확인 프로그램을 실행했다.
□ product_status_result.txt를 확인했다.
□ 정상 / 품절 / 제품없음 / 확인필요 / 확인실패를 구분했다.
□ 품절 상품을 마켓 관리파일에 반영했다.
□ 제품없음 상품을 판매중지 또는 삭제검토로 분류했다.
□ 확인필요 상품을 수동 확인했다.
□ 확인실패 상품은 재실행 또는 수동 확인했다.
```

## 13. 자주 발생할 수 있는 문제

### 문제 1. 상품코드가 검색되지 않음

확인할 것:

```text
상품코드 앞뒤에 공백이 있는가
판매자상품코드를 그대로 넣은 것은 아닌가
도매처 상품코드만 넣었는가
```

---

### 문제 2. 확인필요가 많이 나옴

가능한 원인:

```text
상품코드가 너무 짧음
검색 결과에 유사 코드가 많이 나옴
도매처 검색 방식이 부분검색으로 동작함
```

처리:

```text
확인필요 상품은 수동 확인한다.
짧은 코드는 자동으로 제품없음 처리하지 않는다.
```

---

### 문제 3. 확인실패가 나옴

가능한 원인:

```text
사이트 로딩 지연
로그인 세션 문제
네트워크 문제
도매처 페이지 구조 변경
selector 변경
```

처리:

```text
같은 상품코드만 다시 실행한다.
반복 실패 시 수동 확인한다.
페이지 구조가 바뀐 경우 selector 수정이 필요하다.
```

---

### 문제 4. 프로그램이 끝나지 않고 대기함

가능한 원인:

```text
브라우저 종료 대기 코드가 남아 있음
드라이버 종료 처리가 안 됨
```

처리:

```text
필터링 종료 후 driver.quit()이 실행되는지 확인한다.
브라우저 창을 닫아야 종료되는 구조인지 확인한다.
```

## 14. 품절필터링 결과 활용

품절필터링 결과는 단순 품절처리뿐 아니라 아래 작업에도 활용할 수 있다.

```text
마켓 상품 품절처리
판매중지 대상 선별
삭제검토 대상 선별
그룹상품 대표상품 안정성 확인
리뷰상품 유지 여부 판단
도매처 상품 신뢰도 확인
반복 품절 도매처 확인
```

## 15. 최종 요약

```text
상품코드 추출
→ input_product_codes.txt에 붙여넣기
→ 프로그램 실행
→ product_status_result.txt 확인
→ 정상 / 품절 / 제품없음 / 확인필요 / 확인실패 분류
→ 마켓 관리파일에 반영
```

품절필터링의 핵심은 아래와 같다.

```text
도매처 상품코드를 정확히 넣고
자동 판정 결과를 확인한 뒤
품절·제품없음은 빠르게 처리하고
확인필요·확인실패는 수동 확인하는 것
```
